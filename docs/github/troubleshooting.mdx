## Quick Diagnostics

Having issues with GitHub sync? Start with these quick checks:

<AccordionGroup>
  <Accordion title="✅ Connection Status">
    **Check in Account Settings:**
    - Is your GitHub account connected?
    - Does it show your correct GitHub username?
    - Try disconnecting and reconnecting if uncertain
  </Accordion>

  <Accordion title="✅ Repository Access">
    **Verify in GitHub:**
    - Does the repository still exist?
    - Do you have write access to the repository?
    - Is the repository public or do you own it?
  </Accordion>

  <Accordion title="✅ Sync Settings">
    **Check in Component Settings:**
    - Is GitHub sync enabled for your component?
    - Is there a repository URL displayed?
    - When was the last successful sync?
  </Accordion>
</AccordionGroup>

## Common Issues and Solutions

### Authentication Problems

<Tabs>
  <Tab title="OAuth Connection Failed">
    **Symptoms:**
    - Can't connect GitHub account
    - "Authorization failed" error
    - Redirected to error page

    **Solutions:**
    1. **Clear browser cache** and try again
    2. **Use incognito/private mode** to eliminate cache issues
    3. **Check GitHub login** - ensure you're logged into the correct account
    4. **Disable browser extensions** that might interfere with OAuth
    5. **Try a different browser** if the issue persists

    <CodeGroup>
    ```bash Chrome
    # Clear cache for CodeRocket
    Settings → Privacy → Clear browsing data → Select site data
    ```

    ```bash Firefox
    # Clear site data
    Settings → Privacy → Manage Data → Search for coderocket.app
    ```
    </CodeGroup>
  </Tab>

  <Tab title="Token Expired">
    **Symptoms:**
    - Sync worked before but now fails
    - "Unauthorized" or "Forbidden" errors
    - 401/403 HTTP status codes

    **Solution:**
    1. Go to **Account Settings**
    2. Find **GitHub Integration** section
    3. Click **Disconnect** then **Connect GitHub Account**
    4. Re-authorize CodeRocket with your GitHub account

    <Warning>
      This will require you to re-enable sync for each component, but it will resolve token issues.
    </Warning>
  </Tab>

  <Tab title="Insufficient Permissions">
    **Symptoms:**
    - "Permission denied" errors
    - Can't create repositories
    - Can't push to existing repositories

    **Check GitHub Permissions:**
    1. Go to [GitHub Settings → Applications](https://github.com/settings/applications)
    2. Find "CodeRocket" in Authorized OAuth Apps
    3. Click to view permissions
    4. Ensure these scopes are granted:
       - `repo` (Full repository access)
       - `user:email` (Access email addresses)

    **If permissions are missing:**
    - Revoke the application
    - Reconnect through CodeRocket
    - Grant all requested permissions
  </Tab>
</Tabs>

### Repository Issues

<Tabs>
  <Tab title="Repository Creation Failed">
    **Common causes and fixes:**

    **Repository name already exists:**
    ```bash
    Error: "Repository already exists"
    Solution: Choose a different repository name
    ```

    **Invalid repository name:**
    ```bash
    Error: "Repository name is invalid"
    Solution: Use only letters, numbers, and hyphens
    ```

    **Rate limit exceeded:**
    ```bash
    Error: "API rate limit exceeded"
    Solution: Wait 5-10 minutes and try again
    ```

    **Organization restrictions:**
    ```bash
    Error: "Cannot create repository in organization"
    Solution: Create repository under personal account
    ```
  </Tab>

  <Tab title="Repository Access Lost">
    **If repository was deleted or moved:**

    1. **Check GitHub** - verify repository still exists at expected URL
    2. **Update URL** if repository was renamed or transferred
    3. **Recreate repository** if it was accidentally deleted
    4. **Disconnect and reconnect** GitHub account if permissions changed

    **For transferred repositories:**
    - Repository transfers may break the connection
    - You'll need to disconnect and reconnect GitHub
    - Or create a new repository for the component
  </Tab>

  <Tab title="Wrong Repository Connected">
    **To change which repository a component syncs to:**

    <Warning>
      There's currently no direct way to change repository connections. You'll need to:
    </Warning>

    1. **Disable sync** for the component
    2. **Disconnect GitHub** account (this affects all components)
    3. **Reconnect GitHub** account
    4. **Create new repository** or connect to existing one
    5. **Re-enable sync** for all affected components

    <Info>
      **Future Enhancement**: We're working on allowing repository changes without full disconnection.
    </Info>
  </Tab>
</Tabs>

### Sync Failures

<Tabs>
  <Tab title="Files Not Syncing">
    **Check file types:**
    CodeRocket only syncs code files:
    - ✅ `.js`, `.jsx`, `.ts`, `.tsx`
    - ✅ `.html`, `.css`, `.scss`
    - ✅ `.vue`, `.svelte`
    - ❌ Images, videos, binary files
    - ❌ Node modules, build artifacts

    **Check file sizes:**
    - Maximum file size: 1MB
    - Repository size limit: 100MB total
    - Large files are automatically skipped

    **Check file names:**
    - Avoid special characters: `<>:"|?*`
    - Use forward slashes for paths: `src/components/Button.jsx`
    - Maximum path length: 260 characters
  </Tab>

  <Tab title="Sync Takes Too Long">
    **Normal sync times:**
    - Small projects (< 10 files): 5-15 seconds
    - Medium projects (10-50 files): 15-45 seconds
    - Large projects (50+ files): 45-120 seconds

    **If sync is slower than expected:**
    1. **Check internet connection** - slow upload speeds affect sync time
    2. **Reduce file count** - remove unnecessary files from your component
    3. **GitHub API limits** - may cause delays during peak usage
    4. **Try again later** - temporary GitHub service issues

    <Info>
      CodeRocket shows sync progress in real-time. If it's stuck at a particular step for more than 2 minutes, cancel and try again.
    </Info>
  </Tab>

  <Tab title="Partial Sync Issues">
    **When only some files sync:**

    **Check the sync report:**
    - CodeRocket shows which files were synced vs skipped
    - Skipped files usually have a reason (newer in GitHub, invalid format, etc.)

    **Common scenarios:**
    ```bash
    ✅ Synced: src/App.jsx (updated)
    ⏭️ Skipped: src/utils.js (GitHub version newer)
    ❌ Failed: src/large-file.js (file too large)
    ```

    **To force sync all files:**
    Use "Force Push to GitHub" option, which overrides all checks.
  </Tab>
</Tabs>

### Performance Issues

<Tabs>
  <Tab title="Slow Repository Creation">
    **Typical creation time:** 10-30 seconds

    **If taking longer:**
    - GitHub API may be experiencing delays
    - Large initial file sets take longer to upload
    - Check GitHub status at [status.github.com](https://status.github.com)

    **Optimization tips:**
    - Choose shorter repository names
    - Ensure stable internet connection
    - Avoid peak usage times (business hours in your timezone)
  </Tab>

  <Tab title="Frequent Timeouts">
    **Network-related timeouts:**
    1. **Check internet stability** - intermittent connections cause timeouts
    2. **Disable VPN temporarily** - some VPNs block GitHub API calls
    3. **Corporate firewalls** - may block GitHub API endpoints
    4. **Try mobile hotspot** - to isolate network issues

    **CodeRocket-side timeouts:**
    - Usually indicate temporary service issues
    - Check [CodeRocket status page] for known issues
    - Try again in 5-10 minutes
  </Tab>
</Tabs>

## Advanced Troubleshooting

### Debug Mode

Enable detailed logging to diagnose complex issues:

<Steps>
  <Step title="Open Browser Console">
    - **Chrome/Edge**: F12 → Console tab
    - **Firefox**: F12 → Console tab
    - **Safari**: Develop → Show Web Inspector → Console
  </Step>

  <Step title="Attempt Sync Operation">
    Try the failing sync operation while console is open
  </Step>

  <Step title="Check for Errors">
    Look for error messages starting with:
    - `GitHub API Error:`
    - `Sync failed:`
    - `Repository error:`
  </Step>

  <Step title="Save Error Details">
    Copy any error messages for support tickets
  </Step>
</Steps>

### API Rate Limiting

GitHub has API rate limits that may affect CodeRocket:

<AccordionGroup>
  <Accordion title="Understanding Rate Limits">
    **GitHub API Limits:**
    - **Authenticated requests**: 5,000 per hour
    - **Repository creation**: 300 per hour
    - **File uploads**: No specific limit, but subject to overall rate limit

    **CodeRocket's usage:**
    - Repository creation: 1 request
    - File sync: 1-3 requests per file
    - Repository info: 1 request per sync
  </Accordion>

  <Accordion title="Rate Limit Symptoms">
    **How to identify rate limiting:**
    - Error message mentions "rate limit"
    - HTTP 403 status with rate limit headers
    - Sync fails with "too many requests" error

    **When rate limits reset:**
    - GitHub rate limits reset every hour
    - Wait time shown in error message
    - Usually resolves within 60 minutes
  </Accordion>

  <Accordion title="Avoiding Rate Limits">
    **Best practices:**
    - Don't sync the same component repeatedly in short periods
    - Avoid creating many repositories in quick succession
    - Use "Push to GitHub" instead of "Force Push" when possible
    - Batch your AI iterations rather than making frequent small changes
  </Accordion>
</AccordionGroup>

### Data Consistency Issues

<Tabs>
  <Tab title="Outdated Repository Info">
    **Symptoms:**
    - Repository URL shows but repository doesn't exist
    - Sync reports success but no changes appear in GitHub
    - Component shows "connected" but GitHub shows no CodeRocket commits

    **Fix:**
    1. **Refresh component data**:
       - Go to different component and back
       - Or refresh the browser page
    2. **Verify repository exists** in GitHub
    3. **Check repository permissions** if it exists
    4. **Recreate connection** if data is completely inconsistent
  </Tab>

  <Tab title="Version Mismatches">
    **When CodeRocket and GitHub versions don't match:**

    **Diagnostic steps:**
    1. Check **last sync timestamp** in component settings
    2. Compare with **latest commit date** in GitHub
    3. Look for **sync error messages** in browser console

    **Resolution:**
    - Use "Force Push" to make CodeRocket the source of truth
    - Or use "Pull from GitHub" to bring GitHub changes into CodeRocket
    - Choose based on which version you want to keep
  </Tab>
</Tabs>

### Temporary Workarounds

While waiting for issue resolution:

<Tabs>
  <Tab title="Manual Backup">
    **Export your code manually:**
    1. Copy code from CodeRocket editor
    2. Create files locally or in GitHub manually
    3. Continue development outside CodeRocket temporarily
    4. Use "Pull from GitHub" once sync is working
  </Tab>

  <Tab title="Alternative Sync">
    **Use different sync modes:**
    - If "Push to GitHub" fails, try "Force Push"
    - If automatic sync fails, try manual sync
    - If push fails, try "Pull from GitHub" first
  </Tab>

  <Tab title="New Repository">
    **Start fresh if needed:**
    - Create new repository with different name
    - Disable sync on problematic component
    - Enable sync with new repository
    - Transfer over any important GitHub history manually
  </Tab>
</Tabs>

<Info>
  Most GitHub sync issues are temporary and resolve within an hour. For persistent issues lasting more than 24 hours, please contact support.
</Info>
