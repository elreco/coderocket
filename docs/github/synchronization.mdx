## Automatic Synchronization

When GitHub sync is enabled for a component, CodeRocket automatically pushes every AI-generated version to your repository.

### How it Works

<Steps>
  <Step title="AI Generation">
    You ask the AI to modify your component (e.g., "Make the button blue")
  </Step>

  <Step title="Automatic Commit">
    CodeRocket automatically:
    - Creates a new commit with your changes
    - Uses descriptive commit message: "Update files - Version 3"
    - Pushes to your GitHub repository
  </Step>

  <Step title="Version Tracking">
    Each AI iteration becomes a separate commit, maintaining a complete history of your project's evolution
  </Step>
</Steps>

<Info>
  Automatic sync uses **Force Push** mode to ensure all AI iterations are captured in GitHub, regardless of external changes.
</Info>

## Manual Synchronization Modes

### Pull from GitHub

<Tabs>
  <Tab title="When to Use">
    - You've made changes directly in GitHub
    - Team members have pushed updates
    - You want to incorporate external modifications
    - Merging changes from other development tools
  </Tab>

  <Tab title="How it Works">
    1. Fetches all files from your GitHub repository
    2. Filters code files (`.js`, `.jsx`, `.tsx`, `.html`, `.css`, etc.)
    3. Ignores generated files (`README.md`, `package.json`)
    4. Creates a new version in CodeRocket with GitHub content
    5. Updates the version counter
  </Tab>

  <Tab title="Example Workflow">
    ```
    1. You: "Create a todo app" (Version 1)
    2. GitHub: Team member adds styling
    3. You: Click "Pull from GitHub"
    4. CodeRocket: Creates Version 2 with styling
    5. You: "Add delete functionality" (Version 3)
    ```
  </Tab>
</Tabs>

### Push to GitHub (Smart Sync)

<Tabs>
  <Tab title="When to Use">
    - You want to respect changes made in GitHub
    - Working in a collaborative environment
    - Unsure if GitHub has newer changes
    - Following "Last Write Wins" strategy
  </Tab>

  <Tab title="How it Works">
    1. Compares modification dates between CodeRocket and GitHub
    2. Only pushes if CodeRocket version is newer
    3. Skips files where GitHub is more recent
    4. Shows detailed report of what was synced vs skipped
  </Tab>

  <Tab title="Example Output">
    ```
    ✅ Synced 3 files to GitHub:
    - src/App.jsx (updated)
    - src/components/TodoItem.jsx (new)
    - styles/main.css (updated)

    ⏭️ Skipped 1 file (GitHub version is newer):
    - src/utils/api.js (last modified: 2 hours ago)
    ```
  </Tab>
</Tabs>

### Force Push to GitHub

<Tabs>
  <Tab title="When to Use">
    - You want CodeRocket to be the source of truth
    - Resolving conflicts by choosing CodeRocket version
    - Overriding problematic changes in GitHub
    - Ensuring consistency after major changes
  </Tab>

  <Tab title="How it Works">
    1. Pushes all files regardless of GitHub modification dates
    2. Overwrites any conflicting changes in GitHub
    3. Creates a single commit with all changes
    4. Updates repository to match CodeRocket exactly
  </Tab>

  <Tab title="Use with Caution">
    <Warning>
      Force Push will overwrite any changes made in GitHub. Make sure you've pulled and reviewed GitHub changes before using this option.
    </Warning>
  </Tab>
</Tabs>

## Conflict Resolution Strategy

CodeRocket uses a **"Last Write Wins"** strategy for handling conflicts:

<AccordionGroup>
  <Accordion title="How Timestamps Work">
    - **CodeRocket timestamp**: When the current version was last generated or modified
    - **GitHub timestamp**: When each file was last committed to the repository
    - **Comparison**: File-by-file comparison of modification dates
  </Accordion>

  <Accordion title="Resolution Logic">
    ```
    For each file:
    if (codeRocketTimestamp > githubTimestamp) {
      push file to GitHub
    } else {
      skip file (GitHub is newer)
    }
    ```
  </Accordion>

  <Accordion title="Handling Edge Cases">
    - **New files**: Always pushed to GitHub
    - **Deleted files**: Removed from GitHub if deleted in CodeRocket
    - **Generated files**: `README.md` and `package.json` are always updated
    - **Clock differences**: Small timestamp differences (< 1 minute) are ignored
  </Accordion>
</AccordionGroup>

## Best Practices

### For Solo Development
<Steps>
  <Step title="Enable Automatic Sync">
    Let CodeRocket handle versioning automatically as you iterate with AI
  </Step>

  <Step title="Use Smart Sync for Manual Changes">
    When you make manual edits in GitHub, use "Push to GitHub" to respect your changes
  </Step>

  <Step title="Regular Pulls">
    If you work across multiple environments, regularly pull changes to stay synced
  </Step>
</Steps>

### For Team Collaboration
<Steps>
  <Step title="Establish Workflow">
    Decide whether CodeRocket or GitHub is the primary source of truth for your team
  </Step>

  <Step title="Communicate Changes">
    Let team members know when you're about to make AI iterations that will auto-sync
  </Step>

  <Step title="Use Pull Requests">
    For major changes, consider using GitHub's pull request workflow even with CodeRocket sync
  </Step>
</Steps>

### For External Development
<Steps>
  <Step title="Start with CodeRocket">
    Generate your initial project structure and functionality with AI
  </Step>

  <Step title="Export to GitHub">
    Use Force Push to establish the initial repository state
  </Step>

  <Step title="Develop Externally">
    Continue development in your preferred IDE or with your team
  </Step>

  <Step title="Pull Back When Needed">
    Use "Pull from GitHub" to bring changes back for AI assistance
  </Step>
</Steps>

- **Sync status**: Success, failure, or pending
- **Version synced**: Which CodeRocket version is currently in GitHub

## Troubleshooting Sync Issues

<AccordionGroup>
  <Accordion title="Sync Failed">
    **Possible causes:**
    - Repository was deleted or made private
    - GitHub access token expired
    - Network connectivity issues
    - Repository permissions changed

    **Solutions:**
    - Check repository exists and is accessible
    - Reconnect your GitHub account
    - Verify repository permissions
  </Accordion>

  <Accordion title="Files Not Syncing">
    **Check these items:**
    - File types are supported (code files only)
    - Files are not in ignored patterns
    - Repository has sufficient storage space
    - No special characters in file names
  </Accordion>

  <Accordion title="Timestamp Issues">
    **If timestamps seem incorrect:**
    - Check your system clock settings
    - Consider time zone differences
    - Use Force Push to override timestamp logic
  </Accordion>
</AccordionGroup>
